# Техническое задание: AI-transformation

> **Что мы строим простыми словами:** внутреннюю обучающую платформу для команды IBG, где сотрудники учатся работать с Claude по индивидуальной программе.

---

## Что должна делать платформа

### 1. Регистрация по приглашению
- HR (или Юра) создаёт уникальную ссылку для сотрудника
- Сотрудник по ссылке регистрируется (имя, email, пароль)
- Случайные люди зарегистрироваться **не могут**

### 2. Опрос на 10 вопросов
Сразу после регистрации сотрудник проходит опрос:
- Как зовут?
- Какая роль в IBG? (11 вариантов)
- Опыт работы с AI? (3 уровня)
- Чем конкретно занимаешься? (свободный текст)
- Какие 3-5 задач отнимают больше всего времени?
- С какими инструментами работаешь? (множественный выбор)
- Язык контента? (RU / EN / оба)
- Сколько времени готов выделить в неделю?
- Главная цель обучения?
- Подписка Claude? (Free / Pro / Team / нет)

**Структура опроса детально** в файле `docs/03_ASSESSMENT.json`.

### 3. Автоматическая генерация программы
На основе опроса платформа автоматически выдаёт **персональную программу** из 3-7 курсов.

Логика: **Роль × Уровень AI = шаблон программы**.

11 ролей × 3 уровня = **33 шаблона** уже описаны в SQL-файлах. Вся логика автоматическая, тебе ничего не надо высчитывать вручную — просто запусти SQL-функцию `fn_generate_user_program`.

### 4. Список курсов
Пользователь видит свои курсы в виде карточек:
- Название курса
- Описание (зачем именно ему этот курс)
- Длительность
- Прогресс (% завершения)
- Кнопка "Начать" / "Продолжить"

### 5. Уроки с микро-блоками
Каждый урок состоит из **6 микро-блоков** по 1-15 минут:

```
🎯 Зачем это вам         (1 мин)
🔄 Что у вас сейчас       (3 мин)
📺 Видео + транскрипт     (5-10 мин)
💡 Применение в IBG       (2 мин)
🛠 Микро-практика         (5-10 мин)
✅ Чек-лист самопроверки  (1 мин)
```

В блоке "Применение в IBG" автоматически подгружается **дополнительный кусок контента под роль пользователя**. Например, в уроке про Skills:
- Агент по продажам видит: "Skills для агента: ibg-client-followup, ibg-kp-generator..."
- QC-специалист видит: "Skills для QC: ibg-qc-call-scoring, ibg-qc-feedback..."

### 6. Прогресс
- Каждый урок имеет статус: Не начат / В процессе / Завершён
- Платформа считает потраченное время (heartbeat каждые 30 секунд)
- Виден общий % завершения программы

### 7. Сертификаты
После завершения **всех уроков курса** — автоматически создаётся сертификат:
- Имя пользователя
- Название курса
- Дата
- Уникальный номер (IBG-AI-2026-NNNN)
- Можно скачать (через print-to-PDF браузера)

### 8. Админ-панель
Для Юры и HR — раздел `/admin`:
- Создать invite-ссылку
- Видеть всех пользователей и их прогресс
- Редактировать тексты уроков (без программирования, прямо в браузере)
- Видеть статистику

## Что НЕ делаем в PET-проекте

❌ Email-уведомления — приглашения отправляются вручную через Telegram
❌ PDF-сертификаты — только HTML с возможностью print-to-PDF
❌ Социальные фичи (комменты, лайки) — не нужно
❌ Геймификация (бейджи, очки) — не нужно
❌ Команды/группы (руководитель видит подчинённых) — не нужно
❌ Темная тема — только светлая
❌ Мобильное приложение — только адаптивный веб

Это PET-проект. **Не добавляй фичи, которых нет в этом списке.**

## Структура базы данных

В Supabase 16 таблиц. **Тебе НЕ нужно их создавать руками** — все в SQL-миграциях в папке `migrations/`. Запусти их в SQL Editor и всё.

Кратко что есть:

**Контент:**
- `ob_tracks` — 4 трека (Foundation, AI Fluency, Developer, Enterprise)
- `ob_courses` — 18 курсов
- `ob_modules` — модули внутри курса
- `ob_lessons` — уроки с микро-блоками
- `ob_exercises` — упражнения

**Роли и шаблоны:**
- `ob_roles` — 11 ролей IBG
- `ob_ai_levels` — 3 уровня (newcomer, explorer, practitioner)
- `ob_program_templates` — 33 шаблона программ
- `ob_program_template_courses` — курсы внутри шаблона

**Пользователь:**
- `ob_user_assessments` — ответы на опрос
- `ob_user_programs` — личная программа
- `ob_user_program_courses` — курсы в программе
- `ob_user_progress` — прогресс по урокам
- `ob_certificates` — сертификаты

**IBG-специфика:**
- `ob_lesson_role_overlays` — IBG-инжекции в уроки под роль
- `ob_exercise_role_variants` — варианты упражнений под роль

**Авторизация:**
- `invites` — invite-ссылки

**Функции:**
- `fn_generate_user_program(user_id)` — генерирует программу из шаблона

## Критерии готовности

Твой PET-проект готов к показу Юре, если:

- ✅ Любой по invite-ссылке может зарегистрироваться
- ✅ Опрос проходится за 5-7 минут
- ✅ После опроса появляется персональная программа
- ✅ Курс Claude 101 (12 уроков) проходится от начала до конца
- ✅ Прогресс сохраняется между сессиями
- ✅ Можно открыть на телефоне и пользоваться
- ✅ Админка работает: создание invites, редактирование уроков
- ✅ Сайт развёрнут на Vercel и доступен по URL

Если все галочки — **ты молодец, проект готов**.
