# Решение типовых проблем

> Список типовых ошибок и как их решать самостоятельно.
> **Главное правило:** прежде чем писать Юре — попробуй спросить Claude Code в терминале. В 90% случаев он решит.

## Установка инструментов

### `claude` не найден после установки на Windows

1. Закрой PowerShell полностью
2. Открой заново
3. Проверь: `claude --version`
4. Если всё ещё нет — переустанови по [официальной инструкции](https://docs.claude.com/en/docs/claude-code/setup)

### `node` или `npm` не найдены

1. Зайди на [nodejs.org](https://nodejs.org) и переустанови LTS-версию
2. Перезапусти PowerShell
3. Если всё ещё нет — нужно проверить переменную PATH в Windows. Спроси Claude.ai: "Я установил Node.js на Windows, но в PowerShell команда node не работает. Как починить PATH?"

### PowerShell не выполняет скрипты (ExecutionPolicy)

Открой PowerShell **от имени администратора** и выполни:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
Подтверди "Y".

## Claude Code

### "API key invalid" или "Authentication failed"

```powershell
claude logout
claude login
```
Откроется браузер — авторизуйся заново.

### Claude Code зависает или не отвечает

1. Нажми `Ctrl + C` чтобы прервать
2. Закрой PowerShell, открой заново
3. Запусти `claude` снова

### Claude отказывается делать что-то "опасное"

Это нормально — Claude осторожен с командами вроде `rm -rf` или `drop table`. Если ты уверена, что это нужно — разреши явно: "Я понимаю риски, продолжай".

### Claude теряет контекст между сессиями

Используй файл `CLAUDE.md` в корне проекта. Туда Claude автоматически читает информацию о проекте каждый раз. Спроси его: "Создай CLAUDE.md с описанием моего проекта, чтобы я не объясняла каждый раз".

## Next.js

### `npm run dev` не запускается, ошибки модулей

```powershell
rm -r node_modules
rm package-lock.json
npm install
npm run dev
```

(На Windows вместо `rm -r` используй `Remove-Item -Recurse -Force`.)

### Сайт открывается на другом порту (не 3000)

Это нормально. В терминале смотри строку `Local: http://localhost:XXXX` — открывай этот порт. Обычно 3001, если 3000 занят.

### "Module not found" после установки пакета

Останови сервер (`Ctrl + C`) и запусти `npm run dev` заново. Иногда нужно перезапустить, чтобы Next.js увидел новые модули.

### TypeScript-ошибки в редакторе но `npm run dev` работает

Это конфликт VS Code с TypeScript. Перезапусти TS-сервер: `Ctrl+Shift+P` → "TypeScript: Restart TS Server".

## Supabase

### "JWT expired" или "Invalid JWT"

В `.env.local` неправильные ключи. Зайди в Supabase → Settings → API → скопируй ключи заново.

### Таблицы не создались после SQL-миграции

1. Открой SQL Editor → запусти запрос: `SELECT * FROM ob_courses LIMIT 1;`
2. Если ошибка "relation does not exist" — миграции не запустились
3. Запусти заново все миграции по порядку (001 → 007), смотри ошибки в каждой

### Ошибка "infinite recursion in policy"

В RLS-политиках возникла бесконечная рекурсия. Спроси Claude Code: "В Supabase у меня ошибка 'infinite recursion in policy', как починить?". Обычно нужно изменить SECURITY DEFINER в функции.

### Не могу зарегистрироваться — "User not found in auth"

В Supabase → Authentication → Users — проверь, что пользователь там есть.
Если нет — регистрация не сработала. Проверь, что в `.env.local` правильные ключи.

### `fn_generate_user_program` не работает

```sql
-- Проверь существование функции
SELECT * FROM pg_proc WHERE proname = 'fn_generate_user_program';
```
Если функции нет — миграция 005 не выполнилась полностью. Запусти заново.

## Vercel

### Build падает с ошибкой "Module not found"

Скорее всего у тебя есть локальный файл, который не закоммичен в git. Проверь `git status`. Если файл важный — закоммить его.

### Build падает с ошибкой типизации

В Vercel build строгий — он падает на TS-ошибках, на которые `npm run dev` закрывает глаза. Локально запусти `npm run build` и исправь все ошибки. Если не получается — попроси Claude Code: "У меня build падает с ошибкой [текст]. Помоги починить".

### "Environment Variables not set"

В Vercel → Project → Settings → Environment Variables — проверь, что все 4 переменные добавлены (URL, anon, service_role, app_url). И что отмечены все три окружения: Production, Preview, Development.

### После деплоя сайт показывает 404 на всех страницах кроме главной

Скорее всего проблема с роутингом или middleware. Спроси Claude Code: "После деплоя на Vercel все страницы кроме главной возвращают 404. Помоги диагностировать".

## Git и GitHub

### "Permission denied" при push в GitHub

Тебе нужен GitHub Personal Access Token. Спроси Claude Code: "Я не могу запушить код в свой репозиторий, как настроить аутентификацию через токен на Windows?".

### Закоммитила секреты в Git (например, .env.local)

**СРОЧНО:**
1. **НЕ паникуй**
2. Сразу пиши Юре в Telegram — нужно ротировать ключи
3. Не пытайся сама удалить из истории — это сложно

Урок на будущее: проверяй `.gitignore` перед коммитом. Файл `.env.local` **должен** быть в `.gitignore`.

### Сделала изменения, но `git status` показывает "clean"

Возможно, файлы не сохранены в редакторе. Сохрани все открытые файлы и проверь снова.

## Когда спрашивать Юру

**Не спрашивай Юру:**
- Как установить программу
- Как настроить инструмент
- Какой синтаксис у команды
- Почему что-то не компилируется
- Что делать с ошибкой

Это всё к Claude Code или Claude.ai.

**Спрашивай Юру:**
- Архитектурные решения ("Стоит ли делать Х так или иначе?")
- Когда час не получается решить ОДНУ проблему
- Если случайно закоммитила секреты
- Если что-то связано с инфраструктурой IBG (когда дойдёт до переноса)

## Универсальная стратегия "что-то сломалось"

1. **Сделай скриншот ошибки** или скопируй текст
2. **Спроси Claude Code в терминале:** "У меня ошибка [текст]. Контекст: [что делала]. Помоги диагностировать"
3. **Спроси Claude.ai в браузере** — если в Code не получилось, иногда другая модель видит иначе
4. **Загугли точный текст ошибки** — на StackOverflow часто есть ответ
5. **Только если час не вышло — Юра**
